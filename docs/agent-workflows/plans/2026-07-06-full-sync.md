# Full Sync Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use $subagent-driven-development (recommended) or $executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a full desktop-to-online sync MVP for all current user data, with files stored in Supabase Storage and access limited to the owning authenticated user.

**Architecture:** Add a Supabase-backed snapshot layer for complete local workspaces. The Electron Main Process owns authentication, snapshot creation, Storage upload/download, and local restore; Renderer only shows a non-technical Settings flow over typed IPC.

**Tech Stack:** Electron Main Process, Vue 3 Renderer, TypeScript, SQLite/better-sqlite3, Supabase JS, Supabase Postgres/RLS, Supabase Storage, Vitest.

## Global Constraints

- Nutzertexte dürfen keine technischen Anbieterbegriffe wie Supabase, JSON oder API verwenden.
- Erstellen, Verbinden, Abgleichen und Restore laufen über Modals oder eigene Detailflächen, nicht inline.
- Lokale Daten dürfen nicht still überschrieben werden.
- Dateien liegen in Supabase Storage unter `users/{auth.uid}/workspaces/{local_user_id}/attachments/{attachment_id}/{stored_name}`.
- Storage Policies müssen sicherstellen, dass Nutzer nur eigene Dateien sehen.
- KI-Schlüssel, Entwicklungszugänge und rohe temporäre KI-Entwürfe werden nicht synchronisiert.
